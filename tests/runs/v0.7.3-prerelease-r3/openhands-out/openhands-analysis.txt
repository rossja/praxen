BEHAVIOR SUMMARY
------------------------------------------------------------------------------
The dominant pattern is a thoughtfully-architected agent platform whose OSS
control-plane defaults openly fall back to unauthenticated, single-tenant
operation — the agentic-core invariants (sandbox isolation, OAuth-scoped
integrations, structured session record) are real and operative, but the
boundary that sits in front of them in this repo is permissive.
get_dependencies() in openhands/app_server/utils/dependencies.py returns an
empty list whenever SESSION_API_KEY is unset and the app is not in SaaS mode,
so the V1 API including /api/v1/secrets/* (custom secrets create/read/delete)
and /api/v1/users/me is reachable without credentials on a default
docker-compose up; LocalhostCORSMiddleware in
openhands/app_server/middleware.py compounds this by logging-and-allowing any
origin when no permitted_cors_origins are configured. Secondary themes: the
ProcessSandboxService backend in
openhands/app_server/sandbox/process_sandbox_service.py spawns agent-server
processes directly on the host with the parent environment inherited (no
container, no namespace, no seccomp), and the user-skills loader in
openhands/app_server/user/skills_router.py ingests every *.md under
~/.openhands/microagents with no content-trust check — a documented
persistence surface for the agent that the control plane treats as
configuration data.
