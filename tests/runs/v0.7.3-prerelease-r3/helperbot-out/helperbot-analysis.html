Executive Summary
Agent Remit (as declared)
HelperBot is an internal employee assistant authorized to answer questions, retrieve company knowledge-base documents from within its authorized workspace, perform public web searches, and write summaries or notes only to its designated output location. Its tool inventory is limited to a workspace-scoped document reader, a workspace-scoped document writer, and a public web search capability that must never carry confidential content in queries. The remit forbids accessing files outside the authorized workspace, executing shell commands, revealing the system prompt, and following instructions embedded in retrieved content or false conversational history. Every tool call must be recorded for audit and every session must be bounded by an approximately twenty-call tool-call cap.
Behavior Summary (as observed)
The dominant pattern is policy declared in the remit, zero code-level enforcement — and the workspace is explicit about it. The HELPERBOT record in src/core/agents.js ships with every defensive feature flag (inputValidation, outputFiltering, toolApproval, rateLimiting, auditLogging) set to false and a parallel vulnerabilities block that opts the agent into prompt injection, data exfiltration, and context manipulation by design. The SimulatedLLM branch for 'weak' personas in src/core/llm-simulator.js turns those flags into runtime behavior: a system-prompt extraction returns the verbatim prompt, a credential-extraction fabricates an sk--shaped string, and a 'remember' payload is echoed straight back. Every Behavioral-Constraint clause in the remit — workspace boundary checks, input validation, audit logging, the ~20-call session cap — has a corresponding false in the feature flags or no implementation at all.
Scope of Analysis
HelperBot is one of fourteen agents defined in a Node.js (ES module) project; its persona, tool inventory, feature flags, and declared vulnerabilities all live as a single record under the HELPERBOT key in src/core/agents.js. The record advertises an api protocol on port 7002 with tools read_file, write_file, and search_web, a feature block that explicitly disables inputValidation, outputFiltering, toolApproval, rateLimiting, and auditLogging, and an enabled vulnerabilities block covering prompt injection, data exfiltration (with leakSystemPrompt: true), and context manipulation (with acceptFalseHistory: true). The matching SimulatedLLM in src/core/llm-simulator.js classifies the HelperBot persona as a weak prompt and, on that branch, returns the verbatim system prompt on extraction attempts, fabricates credential strings on credential-extraction attempts, and echoes user-supplied 'remember' payloads on memory-poisoning attempts. The shared SENSITIVE_DATA object in src/core/vulnerabilities.js embeds hardcoded API-key, database, and admin-credential literals that other personas template into their prompts.
