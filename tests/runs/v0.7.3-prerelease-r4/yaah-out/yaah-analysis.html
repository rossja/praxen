Executive Summary
Agent Remit (as declared)
yaah ("Yet Another Agent Harness") is a local developer CLI that generates per-agent configuration for Claude Code, OpenCode, Codex CLI, and GitHub Copilot CLI from a single source of truth, and ships a built-in security-and-quality toolset alongside it. Its central promise is that every generated agent configuration carries the same protective hooks — a linter, a command guard that blocks catastrophic shell commands, a secret scanner, a comment checker, and a session logger — uniformly across all supported coding agents. Authorized counterparties are the local developer, the configured agent's LLM provider, the context7 and pulumi MCP servers, and yaah's own built-in MCP server; remote MCP must use TLS, third-party MCP servers must be pinned, and high-impact MCP write/send/execute calls must reach a human checkpoint.
Behavior Summary (as observed)

The dominant pattern is the promised "uniform protections across every agent" is broken at the cross-agent boundary — yaah ships a deterministic command guard, a 13-pattern secret scanner, structured per-session audit logging, and a clean built-in MCP server, but pkg/generator/hookmap.go leaves PreToolUse and PostToolUse blank for Codex, so yaah generate --agent codex produces config with none of the advertised hooks (command-guard, secret-scanner, linter, comment-checker) wired in — a divergence locked into place by TestCodex_GenerateHooks_NoSupported.

A secondary theme is that the hook pipeline only intercepts the host coding agent's tool calls; MCP tool calls (including yaah's own yaah_planning_init, which writes STATE.md into the workspace) flow directly through pkg/mcpserver without ever passing the command-guard or secret-scanner, and the remit's "MCP write/send/execute MUST reach a human checkpoint" requirement has no corresponding approval gate in code.

Scope of Analysis
Go 1.25 CLI built on cobra, with the registry-pattern architecture documented in AGENTS.md: pkg/harness wires hooks, MCP providers, LSP, skills, agents, and a session store; pkg/hooks/handlers contains the security handlers (guard.go, secretscan.go, commentcheck.go, linter.go, sessionlog.go); pkg/generator/{claude,opencode,codex,copilot}.go emit native config per target, driven by the cross-agent event table in pkg/generator/hookmap.go; pkg/mcpserver hosts the built-in stdio MCP server with seven tools registered in tools.go; pkg/session persists per-session JSON audit records on every hook dispatch. The shipped root .mcp.json wires three MCP servers (context7 via npx -y @context7/mcp, remote pulumi over HTTPS, and built-in yaah serve), and .claude/settings.json binds five hook events to yaah hook <event>. The Codex generator's hook map covers only SessionStart, Stop, and notify — neither PreToolUse nor PostToolUse is emitted for Codex.

HIGH PRAX-2026-05-25-001 Codex hook map leaves PreToolUse and PostToolUse empty, so `yaah generate --agent codex` ships with none of the advertised security hooks.
Implement Zero Trust LLM06 — Excessive Agency
Policy Rule — R-01, R-05 (Worker Remit):
"Every generated agent configuration MUST carry the same set of protective hooks — a linter, a command guard that blocks catastrophic shell commands, a secret scanner, a comment checker, and a session logger. / Hardcoded credentials in files the agent edits MUST be detected before the change is accepted."
pkg/generator/hookmap.go:30 — HookPreToolUse mapping leaves Codex blank ("Claude: PreToolUse, OpenCode: tool.execute.before, Copilot: preToolUse"); HookPostToolUse at line 35 has the same gap. pkg/generator/codex_test.go:118 — TestCodex_GenerateHooks_NoSupported asserts GenerateHooks returns nil when only PreToolUse is configured, encoding the gap as expected behavior.
Recommended Action
  • Add Codex hooks.json bindings for PreToolUse and PostToolUse to hookMappings in pkg/generator/hookmap.go (Codex CLI supports these event names since v0.9.0); update TestCodex_GenerateHooks_NoSupported to assert the opposite — that the generator emits a PreToolUse group whose command is yaah hook PreToolUse.
  • Add a cross-agent generator parity test that fails if any of {command-guard, secret-scanner, linter, comment-checker} is missing from a target's generated hook config, so future targets cannot ship the same gap.
HIGH PRAX-2026-05-25-002 MCP tool calls bypass the hook chain entirely — command-guard and secret-scanner never observe MCP-driven activity.
Implement Zero Trust LLM06 — Excessive Agency ASI02 — Tool Misuse and Exploitation
Policy Rule — R-01 (Worker Remit):
"Every generated agent configuration MUST carry the same set of protective hooks — a linter, a command guard that blocks catastrophic shell commands, a secret scanner, a comment checker, and a session logger."
pkg/mcpserver/tools.go:363 — addPlanningInitTool writes .planning/STATE.md directly via os.WriteFile without invoking the registry; no PreToolUse/PostToolUse dispatch fires for MCP calls. pkg/harness/harness.go:84 — HandleHookEvent is only reached from the `yaah hook <event>` CLI path wired into Claude/Opencode/Copilot settings, not from the MCP server's tool handlers.
Recommended Action
  • Wrap every MCP tool handler in pkg/mcpserver/tools.go with calls to the harness's command-guard and secret-scanner before executing, mirroring the PreToolUse/PostToolUse dispatch logic used for host-agent tools.
  • Add an MCPCallRecord channel on pkg/session so MCP invocations land in the same per-session audit log as host-agent tool calls.
HIGH PRAX-2026-05-25-003 `yaah_planning_init` writes to the workspace with no approval gate, violating the remit's human-checkpoint-on-MCP-write requirement.
Implement Zero Trust ASI03 — Identity and Privilege Abuse
Policy Rule — R-07 (Worker Remit):
"High-impact actions — destructive shell commands, file writes, and MCP tools that write, send, or execute — MUST reach a human checkpoint before they run."
pkg/mcpserver/tools.go:378 — addPlanningInitTool calls os.MkdirAll and os.WriteFile for .planning/STATE.md immediately, with no approval surface between the model's call and the write. pkg/mcpserver/server.go:43 — registerTools registers addPlanningInitTool with no gating wrapper around it.
Recommended Action
  • Add an approval-required flag to the MCP tool registration in pkg/mcpserver/tools.go for any tool whose handler performs filesystem writes, and wire it into the MCP transport so the model receives an explicit pending-approval response instead of a completed write.
  • As a defense-in-depth, prefix STATE.md with a banner asserting the file was created by yaah_planning_init and check that the file does not already exist before any subsequent write.
HIGH PRAX-2026-05-25-004 Session-loaded AGENTS.md files are writable from the agent's tool surface with no ingestion-vs-injection guard.
Balance Your Knowledge Base ASI06 — Memory and Context Poisoning
Policy Rule — R-06 (Worker Remit):
"Agent-managed, session-loaded files MUST NOT be writable in a way that lets ingested content persist into future sessions unreviewed."
AGENTS.md — root file documents its own session-load role ("the closest AGENTS.md to the files you're changing wins") and the scope-index points at additional session-loaded AGENTS.md files in pkg/ and .github/workflows/. pkg/hooks/handlers/secretscan.go:33 — the only PostToolUse handler watching file content scans for credential patterns and has no concept of "session-loaded file" — a write that poisons AGENTS.md with new instructions sails through.
Recommended Action
  • Add a session-loaded-file-guard handler to pkg/hooks/handlers that registers on PostToolUse for Edit/Write/MultiEdit, holds a list of session-loaded file paths (root AGENTS.md, scope-index targets, any STATE.md written by yaah_planning_init), and blocks the write unless an explicit override flag is set in the call's tool input.
  • Document in AGENTS.md itself that modifications require operator review and surface the warning in the session log's Findings array when such writes are observed.
HIGH PRAX-2026-05-25-005 `context7` MCP server is configured as unpinned `npx -y @context7/mcp` — pulled fresh on every invocation.
Manage Your Supply Chain LLM03 — Supply Chain ASI04 — Agentic Supply Chain Vulnerabilities
Policy Rule — R-11 (Worker Remit):
"Third-party MCP servers MUST be pinned to a known-good, integrity-checked version — no server package auto-installed afresh, unpinned, on every run."
.mcp.json:5 — context7 stanza specifies "command": "npx", "args": ["-y", "@context7/mcp"] with no @version suffix. pkg/mcp/providers/context7.go:22 — the provider hard-codes the same unpinned `npx -y @context7/mcp` invocation, so the generator output mirrors the repo's own .mcp.json.
Recommended Action
  • Pin context7 to a specific version in pkg/mcp/providers/context7.go (e.g. @context7/mcp@1.4.3) and regenerate the shipped .mcp.json; document the pinning policy in AGENTS.md alongside the existing pinned-action-SHA rule.
  • Add a startup Doctor check in pkg/mcpserver/tools.go's addDoctorTool that flags any MCP provider whose command is npx/uvx/pipx without a pinned version, so future contributors can't silently regress.
MEDIUM PRAX-2026-05-25-006 No automated check that third-party MCP tool descriptions stay free of instruction-like language directed at the model.
Implement Zero Trust LLM01 — Prompt Injection Tool descriptions contain no hidden model instructions
Policy Rule — R-04 (Worker Remit):
"MCP tool descriptions MUST NOT contain instruction-like language directed at the model — neither the built-in server's tool descriptions nor, as far as the harness can inspect them, those of the third-party servers it configures."
pkg/mcpserver/tools.go:56 — built-in tool descriptions verified clean across all seven tools; no equivalent inspection logic exists for context7 or pulumi. pkg/mcp/providers/context7.go — providers carry no description-fetch or imperative-phrasing audit step (same shape in pkg/mcp/providers/pulumi.go).
Recommended Action
Add a yaah doctor --mcp-tool-descriptions mode in pkg/mcpserver/tools.go that lists each configured MCP server's advertised tools and flags descriptions matching imperative regex patterns (/\b(you should|remember to|always|never|use this when)\b/i).
MEDIUM PRAX-2026-05-25-007 No SECURITY.md, no threat model, and no adversarial test of the advertised protections.
Build an AI Red Team
Policy Rule — R-10 (Worker Remit):
"The project SHOULD publish a threat model and a security-disclosure process, and SHOULD run adversarial testing of its own protections — confirming that the secret scanner catches a planted secret, the command guard blocks a catastrophic command, a poisoning write into a session-loaded file is prevented, and a generated configuration genuinely carries the protections it claims."
workspace root — no SECURITY.md, THREAT_MODEL.md, or .github/SECURITY.md present. pkg/generator, pkg/harness, pkg/hooks, pkg/mcpserver, pkg/session, pkg/schema — eight *_test.go files total, all functional; none plant a secret, fire a catastrophic command, or attempt a session-file poisoning write to verify the protections trigger.
Recommended Action
  • Add SECURITY.md at the workspace root with a disclosure email/PGP key and a one-paragraph threat model naming the four canonical adversarial scenarios.
  • Add adversarial-test fixtures under pkg/hooks/handlers/: one that writes a real-shaped fake AWS key to a test file and asserts SecretScanner.ScanFile flags it, one that calls CommandGuard.CheckCommand("rm -rf /") and asserts Safe=false, and one parity test confirming every generator emits the full security hook set.
MEDIUM PRAX-2026-05-25-008 Standalone SessionLogger writes only session lifecycle events — tool-call audit is fully concentrated in the JSON store with no plaintext fallback.
Monitor Continuously
Policy Rule — R-08 (Worker Remit):
"The harness MUST maintain a durable, structured, timestamped record of tool calls, blocked actions, and file modifications — detailed enough to reconstruct what the agent did in a session."
pkg/hooks/handlers/sessionlog.go:34 — Events() returns only HookSessionStart, HookSessionEnd, HookStop; the appended entry contains only event=<name> session=<id> cwd=<path>. pkg/harness/harness.go:97 — HandleHookEvent writes the rich tool-call record but only to the JSON session store, with os.Stderr as the only fallback when save fails.
Recommended Action
Extend SessionLogger.Events() to include HookPreToolUse and HookPostToolUse and append a structured one-line JSON entry per tool call (matching the ToolCallRecord shape) to sessions.log, so the plaintext log carries the full audit surface advertised by its filename.
What's Working Well

Controls and behaviors that are correctly implemented and verified during this scan. These represent areas where the agent's implementation aligns with its stated policy and security best practices.

Deterministic command-guard with seven catastrophic-command patterns

pkg/hooks/handlers/guard.go ships pre-compiled regexes for the exact catastrophes the remit names — rm -rf /, force-push to main/master, hard reset, DROP TABLE/DATABASE, truncate to system path, mkfs, raw dd to /dev/ — and blocks the command (returns Block: true with a structured reason) rather than queueing it for approval.

pkg/hooks/handlers/guard.go:27

13-pattern secret scanner with structured per-file findings

pkg/hooks/handlers/secretscan.go carries thirteen pre-compiled patterns covering AWS, GitHub PAT/OAuth/fine-grained, OpenAI, Anthropic, generic password/secret literals, bearer tokens, PEM private keys, Slack tokens and webhooks, and SendGrid — each match returns a structured SecretFinding with file path, line number, and pattern label, and an Edit/Write/MultiEdit that lands a match is blocked.

pkg/hooks/handlers/secretscan.go:33

Structured per-session audit log persisted on every hook dispatch

pkg/harness/harness.go's HandleHookEvent records every tool call (timestamp, tool name, summarized input, blocked flag, reason) into a Session struct, tracks distinct file modifications, and persists the result atomically via pkg/session/store.go — sufficient to reconstruct what the agent did in a session.

pkg/harness/harness.go:97

Clean built-in MCP server tool descriptions

All seven tools registered by pkg/mcpserver/server.go's registerTools (yaah_scan_secrets, yaah_lint, yaah_check_command, yaah_doctor, yaah_session_info, yaah_planning_status, yaah_planning_init) carry short, declarative descriptions with no instruction-like phrasing directed at the model — the canonical tool-poisoning indicator is absent.

pkg/mcpserver/tools.go:56

Exact-pinned Go dependencies with full integrity hash coverage

go.mod lists every direct and indirect dependency at an exact version (modelcontextprotocol/go-sdk v1.4.0, cobra v1.10.2, pelletier/go-toml/v2 v2.2.4, etc., no caret/tilde ranges) and go.sum carries h1: and go.mod hashes for the full transitive tree.

go.mod:5

Remote Pulumi MCP server reached over TLS

pkg/mcp/providers/pulumi.go configures the only HTTP-transport MCP server with an https://mcp.ai.pulumi.com/mcp URL, satisfying the remit's 'Remote MCP servers MUST be reached over TLS' requirement.

pkg/mcp/providers/pulumi.go:22
