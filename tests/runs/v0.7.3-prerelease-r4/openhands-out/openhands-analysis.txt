BEHAVIOR SUMMARY
------------------------------------------------------------------------------
The dominant pattern is declared-but-not-enforced auth on the OSS application
server: openhands/app_server/utils/dependencies.py's get_dependencies()
returns an empty list of FastAPI dependencies whenever SESSION_API_KEY is
unset and app_mode is not SAAS — the default OSS configuration — so the entire
/api/v1 surface (conversations, sandbox lifecycle, settings, secrets including
stored git-provider tokens, skills, git operations) is open to any caller that
can reach the port. The router comments throughout the OSS code candidly admit
the actual protection is provided by SetAuthCookieMiddleware, but that
middleware lives only in enterprise/saas_server.py and is never added in
openhands/app_server/app.py. The compound risk: LocalhostCORSMiddleware falls
open when no origins are configured (it explicitly logs a warning and returns
True for any origin), the ProcessSandboxService backend runs agent code
unisolated on the host, and skills/microagents are loaded from local files
with no content-trust check — so an unauthenticated network attacker who can
reach the listener has a one-hop path from /api/v1/secrets token theft to
host-process code execution through a planted skill.
