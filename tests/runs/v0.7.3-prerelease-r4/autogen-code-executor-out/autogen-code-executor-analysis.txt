BEHAVIOR SUMMARY
------------------------------------------------------------------------------
The dominant pattern is a sandbox that has the shape of isolation but not the
substance: Docker and Azure are available paths, but the defaults the executor
accepts make every path a sharp tool. create_default_code_executor() silently
downgrades to LocalCommandLineCodeExecutor on any Docker failure (only a
warnings.warn to stderr is interposed), LocalCommandLineCodeExecutor.__init__
itself merely warns — never gates — and copies the host's full os.environ into
every child process, and DockerCommandLineCodeExecutor.start constructs the
container with no user=, read_only=, mem_limit=, cap_drop=, pids_limit=, or
network_mode="none" argument while bind-mounting the work directory as rw.
Layered on top, no executor records a per-execution audit entry (timestamp,
source agent, code digest, exit status) — logging.info/debug calls only cover
infrastructure events like image pulls and container stop — so the remit's
audit clause has no implementation anywhere in the subsystem.

CRITICAL FINDINGS

  PRAX-2026-05-25-001  <code>LocalCommandLineCodeExecutor</code> proceeds with
            host-level code execution after only a <code>warnings.warn</code>
            — no approval gate, no production-mode rejection.
      Action 1: Replace the warnings.warn call at local/__init__.py:163 with a
                constructor argument require_approval: bool (default True)
                that raises RuntimeError unless the caller passes
                acknowledged_local_execution=True and an
                AUTOGEN_LOCAL_EXECUTOR_APPROVED environment variable is set.
      Action 2: Refuse to start when the process detects production indicators
                (e.g., os.environ.get("AUTOGEN_ENV") == "production")
                regardless of the flag.

  PRAX-2026-05-25-002  <code>LocalCommandLineCodeExecutor</code> copies the
            host's full <code>os.environ</code> into every executed
            subprocess, exposing all host secrets to LLM-authored code.
      Action 1: Replace env = os.environ.copy() with a minimal allowlist

                LocalCommandLineCodeExecutor docstring at the danger
