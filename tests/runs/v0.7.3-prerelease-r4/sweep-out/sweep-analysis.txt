BEHAVIOR SUMMARY
------------------------------------------------------------------------------
The dominant pattern is policy declared in the remit, never enforced in
scope-readable code: every untrusted-input rule in the Behavioral Constraints
section (R-08, R-09, R-15) has zero corresponding implementation — issue
bodies, PR comments, and file diffs flow straight into Anthropic messages
arrays with no neutralization, no quote-wrapping, and no instruction-pattern
detector. That gap meets a second pattern of control primitives present in the
codebase but wired out: WEBHOOK_SECRET is declared in config/server.py and
consumed by verify_signature, but the verifier returns True when the secret is
unset (fail-open), and is_blocked() is defined in sweep_bot.py with no caller
anywhere in the repo. Compounding both is a row of four subprocess.run(...,
shell=True) sites whose arguments include LLM-output or untrusted-issue text —
the most direct of these is the ripgrep tool in question_answerer.py, where
the model's chosen search query is " ".join-ed into a shell command line with
no escaping. Together these form a credible injection-to-shell chain that the
remit explicitly forbids (R-14, R-15).
