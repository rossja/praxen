Executive Summary
Agent Remit (as declared)
Sweep is an autonomous software-maintenance agent that reads GitHub issues and comments, generates code changes in response, and opens pull requests for human review. It operates as a GitHub App scoped to specific repositories; permitted counterparties are GitHub (the installed repos), the LLM provider (for inference), the issue/PR author (whose text is untrusted input), and an internal job queue. Sweep must treat every character of an issue, comment, file, or diff as untrusted, must never merge its own PRs, must never execute arbitrary shell commands or run generated code, must never modify CI/CD or security-sensitive files without explicit human approval, and must never communicate outside GitHub other than to its LLM provider.
Behavior Summary (as observed)

Sweep's declared isolation model — "treat all issue/PR text as untrusted; never execute arbitrary shell; never communicate outside GitHub and the LLM provider; never merge" — is contradicted at the code level by a chain of un-gated execution and exfiltration primitives reachable from issue/comment text. The codegen pipeline is full of subprocess.run(" ".join([...query...]), shell=True) wrappers (core/context_pruning.py:186, agents/question_answerer.py:270, config/client.py:338, utils/code_validators.py:303/512/585) whose query/filename arguments originate in untrusted LLM output or repository paths — direct command injection from the same channels the remit names as untrusted.

The WEBHOOK_SECRET path silently bypasses signature verification when the env var is unset (utils/hash.py:22-24, returns True), the /jira endpoint and the /update_sweep_prs_v2 GET both take untrusted input with no auth, and there is no code-level prompt-injection screening of issue/comment bodies before they enter the model context — every "MUST be screened for instruction injection" clause in the remit lacks a corresponding control. Outbound destinations (PostHog, Loki, Sentry, Discord, Keygen) expand well beyond the remit's GitHub-plus-LLM allowance.

Scope of Analysis
Python FastAPI service in sweepai/api.py with a mounted chat backend at sweepai/chat/api.py; webhook handlers (on_ticket, on_comment, review_pr, on_check_suite, on_jira_ticket) dispatch into a multi-agent codegen pipeline under sweepai/agents/ and sweepai/core/ (modify, sweep_bot, context_pruning, question_answerer, search_agent). Configuration lives in sweepai/config/server.py (env-loaded literals; declares a WEBHOOK_SECRET, a DOCKER_ENABLED flag defaulting to false, hardcoded PostHog write key) and sweepai/config/client.py. The agent ships multiple subprocess.run(... shell=True) call sites (ripgrep wrappers, github-linguist, ESLint/Prettier, Docker build/run in on_check_suite.py) and an html_extractor that fetches arbitrary URLs via requests.get. Outbound destinations beyond the remit's allowed set include PostHog (analytics), Loki, Sentry, Discord status webhooks, and a Keygen license API.
