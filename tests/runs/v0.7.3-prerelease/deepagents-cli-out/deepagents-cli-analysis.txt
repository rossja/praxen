BEHAVIOR SUMMARY
------------------------------------------------------------------------------
The CLI's central guarantee — that an open API cannot be deployed without
explicit operator confirmation — is implemented as a single boolean in
deploy/commands.py (is_anonymous) that fires the warning + [y/N] gate only
when both [frontend].enabled and [auth].provider == "anonymous". A project
that sets [auth] provider = "anonymous" without [frontend] still emits the
permissive auth.py (overriding LangSmith Cloud's default x-api-key) and ships
an open API with no warning and no confirmation, breaking the remit's core
promise. Around that gap, the bundler is otherwise disciplined — .env is
excluded from _seed.json, dry-run never contacts the hub or langgraph deploy,
MCP stdio transports are rejected for deploy, the frontend config injection
escapes <, and unit tests cover the anonymous abort paths — but MCP URLs are
not constrained to TLS, remote MCP versions are not pinned, and dev --host
0.0.0.0 exposes the auth-disabled dev server with only an informational print.
