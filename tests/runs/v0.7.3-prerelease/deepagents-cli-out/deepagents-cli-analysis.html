Executive Summary
Agent Remit (as declared)
The Deep Agents CLI is a local developer tool that scaffolds, bundles, and ships a deepagents agent project — init, dev, and deploy are its only working surfaces. Its central promise is that the deployed bundle faithfully reflects the developer's reviewed project: bundling is restricted to declared sources (configuration, system prompt, skills, declared MCP servers, subagents), remote MCP servers must be reached over TLS and validated, and any deploy that would leave the deployed API open without authentication must require explicit operator confirmation. Credentials must never be embedded into seeded payloads or printed/logged.
Behavior Summary (as observed)

The CLI's central guarantee — that an open API cannot be deployed without explicit operator confirmation — is implemented as a single boolean in deploy/commands.py (is_anonymous) that fires the warning + [y/N] gate only when both [frontend].enabled and [auth].provider == "anonymous". A project that sets [auth] provider = "anonymous" without [frontend] still emits the permissive auth.py (overriding LangSmith Cloud's default x-api-key) and ships an open API with no warning and no confirmation, breaking the remit's core promise.

Around that gap, the bundler is otherwise disciplined — .env is excluded from _seed.json, dry-run never contacts the hub or langgraph deploy, MCP stdio transports are rejected for deploy, the frontend config injection escapes <, and unit tests cover the anonymous abort paths — but MCP URLs are not constrained to TLS, remote MCP versions are not pinned, and dev --host 0.0.0.0 exposes the auth-disabled dev server with only an informational print.

Scope of Analysis
A Python argparse-driven CLI under libs/cli/deepagents_cli/ with three top-level subcommands wired in main.py and implemented in deploy/commands.py. Bundling lives in deploy/bundler.py, TOML config parsing/validation in deploy/config.py, and generated artifacts (deploy_graph.py, auth.py, _seed.json, _mcp.json, langgraph.json, pyproject.toml) are emitted from deploy/templates.py. The deploy path shells out to the external langgraph CLI for the actual push and runs langgraph dev for the local dev server (bound to 127.0.0.1 by default, optionally 0.0.0.0). A LangSmith Hub backend (deploy/context_hub.py) is invoked at bundle time when [memories].backend = "hub". A published THREAT_MODEL.md and a comprehensive unit / integration test tree accompany the package.

HIGH PRAX-2026-05-23-002 MCP config validator does not enforce TLS — plaintext http:// URLs pass and are carried into the deployed bundle
Implement Zero Trust LLM03 — Supply Chain ASI04 — Agentic Supply Chain Vulnerabilities
Policy Rule — R-04 (Worker Remit):
"Remote MCP servers carried into the bundle MUST be reached over TLS, and their transport configuration MUST be validated before the bundle is shipped."
libs/cli/deepagents_cli/deploy/config.py:281 — _validate_mcp_for_deploy iterates servers and only checks transport != 'stdio' (lines 281-301); no inspection of the url field for scheme libs/cli/deepagents_cli/deploy/config.py:294 — transport = server_config.get('type', server_config.get('transport', 'stdio')) — only 'stdio' is rejected; an http:// URL with transport='http' passes silently libs/cli/deepagents_cli/deploy/bundler.py:194 — mcp_present check + shutil.copy2(project_root / MCP_FILENAME, build_dir / '_mcp.json') — config is copied verbatim into the bundle (lines 194-197)
Recommended Action

  • Mirror the same scheme check in the generated _load_mcp_tools runtime template (deploy/templates.py:410-481) so a hostile ${VAR} substitution at runtime cannot downgrade to plaintext.
HIGH PRAX-2026-05-23-003 Remote MCP servers carried into the bundle are not pinned to a known-good integrity-checked version
Manage Your Supply Chain LLM03 — Supply Chain ASI04 — Agentic Supply Chain Vulnerabilities
Policy Rule — R-21 (Worker Remit):
"Remote MCP servers declared in the project MUST be pinned to a known-good, integrity-checked version, so the deployed agent does not auto-install an unpinned server afresh."
libs/cli/deepagents_cli/deploy/config.py:281 — _validate_mcp_for_deploy: only stdio is rejected; no version pin, content hash, or integrity field is inspected (lines 281-301) libs/cli/deepagents_cli/deploy/templates.py:444 — MCP_TOOLS_TEMPLATE _load_mcp_tools: builds connections from the bundled _mcp.json with no version assertion; whatever the URL serves at runtime is what the agent loads (lines 444-481) .mcp.json — Top-level project .mcp.json declares two MCP servers (docs-langchain, reference-langchain) with bare https URLs and no version field — illustrating the schema the validator accepts

What's Working Well

Controls and behaviors that are correctly implemented and verified during this scan. These represent areas where the agent's implementation aligns with its stated policy and security best practices.

Dry-run does not contact mutating endpoints

_deploy returns immediately after print_bundle_summary when --dry-run is set, never calling _seed_hub_repo or _run_langgraph_deploy; the anonymous-deploy confirmation prompt is also skipped on dry-run by design.

libs/cli/deepagents_cli/deploy/commands.py:335-338

.env excluded from the bundled seed payload

_build_seed walks only skills/, user/, subagents/, and AGENTS.md; .env is copied separately as a sibling of _seed.json and is never folded into the memories or skills payloads.

libs/cli/deepagents_cli/deploy/bundler.py:200-206

Frontend config injection escapes script-tag breakout

_build_runtime_config_json JSON-dumps the runtime config and replaces every < with \u003c so a hostile or accidental </script> in a string value cannot break out of the inline <script> tag.

libs/cli/deepagents_cli/deploy/bundler.py:126-128

Scaffold overwrite requires explicit --force

_init_project refuses to scaffold over an existing directory unless --force is passed; the unit-test suite covers both the refusal and the opt-in.

libs/cli/deepagents_cli/deploy/commands.py:197-199

Strict TOML schema rejects unknown sections and keys

_parse_config rejects any top-level TOML section outside {agent, sandbox, auth, memories, frontend} and any unknown key in each section, preventing silent absorption of out-of-band configuration.

libs/cli/deepagents_cli/deploy/config.py:447-479

Deep unit-test coverage of the anonymous-mode warning and abort paths

tests/unit_tests/deploy/test_frontend_bundle.py covers the warning text, the abort-on-no flow, the abort-on-EOF flow, and the dry-run-skips-confirmation behavior — a substantive red-team-style coverage of the gate that exists today.

libs/cli/tests/unit_tests/deploy/test_frontend_bundle.py:276-377
