Executive Summary
Agent Remit (as declared)
OpenHands is an autonomous software-engineering agent that accepts natural-language tasks and completes them end-to-end by writing code, executing it in a sandboxed runtime, browsing the web, and operating on connected git/issue-tracker integrations. The remit treats all task prompts, web pages, issue content, repository files, and micro-agent or memory content as untrusted input, requires every agent-generated command to execute inside the sandbox and every file access to stay inside the per-session workspace, and forbids destructive integration operations (repo deletion, webhook removal, collaborator revocation) and cross-session state sharing entirely. Several actions — writes to a foreign repository, PR merge, force-push/branch-delete/history-rewrite, dependency commits, CI/CD edits, adding an MCP server at runtime — require explicit user confirmation, and every action must land in a durable session record with per-session wall-clock and step-count caps enforced.
Behavior Summary (as observed)

The dominant pattern in this scan is auth-by-environment-variable: every protected V1 router declares dependencies=get_dependencies(), but get_dependencies() in app_server/utils/dependencies.py only attaches a real check when the SESSION_API_KEY env var is set. In OSS mode with no env var (the default for a developer running docker compose up), the entire /api/v1 surface — sandbox start/delete, secrets CRUD, settings, conversation create — is reachable with no authentication, and the DefaultUserAuth implementation returns None for user_id, collapsing every caller into a single shared identity.

The two reinforcing weaknesses are the localhost-CORS middleware in app_server/middleware.py, which logs a warning and allows any origin when permitted_cors_origins is empty (the default), and the InMemoryRateLimiter attached at 10 req/s — a soft mitigation that does not constitute access control. The remit's authentication, per-session isolation, and approval-gate clauses (cross-repo writes, PR merge, MCP-server-add) have no code-level enforcement in the default OSS deployment; the agent relies entirely on the operator's deployment perimeter to keep these endpoints off the network.

Scope of Analysis
FastAPI application factored across openhands/app_server/ (the V1 surface, primary in-scope code) and openhands/server/ (a deprecated V0 shim that re-exports the V1 app). The V1 app composes a global v1_router mounted at /api/v1 with sub-routers for conversations, sandboxes, secrets, settings, git, webhooks, and an MCP server proxied under /mcp, plus an SPA static mount, a localhost-permissive CORS middleware, an in-memory rate limiter, and a no-cache header middleware. Sandbox execution is pluggable across Docker, process-spawn, and remote-runtime backends; integration credentials and per-user state are kept in pluggable secrets/settings stores (default: local JSON files). The single auth gate for the entire /api/v1 surface is the optional SESSION_API_KEY env var consumed in app_server/utils/dependencies.py — when unset the OSS deployment exposes all V1 endpoints unauthenticated.

HIGH PRAX-2026-05-23-001 Entire /api/v1 surface is unauthenticated by default in OSS mode — no <code>SESSION_API_KEY</code> means <code>get_dependencies()</code> attaches no auth check.
Implement Zero Trust ASI03 — Identity and Privilege Abuse
Policy Rule — R-05 (Worker Remit):
"Integration and session credentials MUST be verified on every request to an integration, and long-lived credentials MUST NOT be cached anywhere the model can reach."
openhands/app_server/utils/dependencies.py:23 — get_dependencies() returns [] when SESSION_API_KEY is unset and app_mode != SAAS; routers wire this as their only auth dependency. openhands/app_server/utils/dependencies.py:9 — _SESSION_API_KEY = os.getenv('SESSION_API_KEY') — read once at import time; an unset env var disables auth for the entire process lifetime. openhands/app_server/sandbox/sandbox_router.py:32 — router = APIRouter(prefix='/sandboxes', tags=['Sandbox'], dependencies=get_dependencies()) — same pattern in secrets_router, settings_router, app_conversation_router, git_router.
Recommended Action
  • Make a real authentication gate the default for OSS mode — either require SESSION_API_KEY at startup (fail-closed if unset) or generate and persist one on first run and surface it in the startup log.
  • Document in app_server/README.md that an OSS deployment with no SESSION_API_KEY exposes the V1 API unauthenticated, and that this configuration is suitable only for a loopback-only bind.
  • Add a startup check that refuses to bind a non-loopback interface when no auth is configured.
HIGH PRAX-2026-05-23-002 CORS middleware silently allows <strong>any</strong> origin when <code>permitted_cors_origins</code> is empty — the default OSS configuration.
Implement Zero Trust LLM06 — Excessive Agency
openhands/app_server/middleware.py:34 — is_allowed_origin: if not self.allow_origins and not self.allow_origin_regex: ... return True — emits a warning and allows the request rather than denying. openhands/app_server/middleware.py:28 — allow_credentials=True is paired with the open-by-default origin rule; cookies and the X-Session-API-Key header are accepted from any origin in this state. openhands/app_server/config.py:110 — get_default_permitted_cors_origins() returns [] when neither OH_PERMITTED_CORS_ORIGINS_* nor PERMITTED_CORS_ORIGINS is set — the OOTB OSS deployment.
Recommended Action
  • Change the empty-origins branch of LocalhostCORSMiddleware.is_allowed_origin to allow only localhost/127.0.0.1 (the existing first branch), and reject all other origins with a denial instead of a warning.
  • Make OH_PERMITTED_CORS_ORIGINS_0 required when the server binds a non-loopback interface.

Implement Zero Trust ASI03 — Identity and Privilege Abuse
Policy Rule — R-13 (Worker Remit):
"One session's state, memory, or credentials MUST NOT leak into another session."
openhands/app_server/user_auth/default_user_auth.py:32 — async def get_user_id(self): return None — comment: 'does not support multi tenancy, so user_id is always None'. openhands/app_server/secrets/file_secrets_store.py:37 — FileSecretsStore.get_instance(user_id) ignores user_id and uses the single global file_store; same physical file (~/.openhands/secrets.json) serves every caller. openhands/app_server/user_auth/default_user_auth.py:99 — get_for_user(cls, user_id): assert user_id == 'root' — every off-DefaultUserAuth code path that asks for a user expects exactly 'root'.

HIGH PRAX-2026-05-23-004 <code>ProcessSandboxService</code> launches the agent server as a host subprocess with the parent's environment — "sandbox" in name only.
Implement Zero Trust ASI05 — Unexpected Code Execution (RCE)
Policy Rule — R-01 (Worker Remit):
"All agent-generated code MUST execute inside the sandboxed runtime and never directly on the host."
openhands/app_server/sandbox/process_sandbox_service.py:120 — env = os.environ.copy(); env.update(sandbox_spec.initial_env) — the parent's full environment, including any cloud-provider credentials, is exported into the child. openhands/app_server/sandbox/process_sandbox_service.py:141 — subprocess.Popen(cmd, env=env, cwd=working_dir, ...) — no chroot, no namespaces, no seccomp, no rlimits, no UID drop; the child runs as the OpenHands process owner. openhands/app_server/config.py:341 — config.sandbox = ProcessSandboxServiceInjector() is selected for RUNTIME=local or RUNTIME=process — the user-facing knob doesn't surface the isolation downgrade.
Recommended Action
  • Document on the ProcessSandboxService docstring and in the runtime-selection README that this backend provides no isolation from the host and is intended for trusted CI/dev contexts only.
  • Refuse to set RUNTIME=local|process in the same deployment where any untrusted-content channel is enabled (web browser, public repos, issue webhooks).
  • If a lightweight in-process sandbox is needed, add at minimum a fresh env (allowlist of env vars), a temporary cwd outside the operator's home, and a non-zero UID drop on POSIX hosts.

HIGH PRAX-2026-05-23-009 Action-level audit logging in the app-server is Python stdlib loggers to stderr — no durable, structured, security-relevant session record as the remit requires.
Monitor Continuously
Policy Rule — R-06 (Worker Remit):
"Every action the agent takes — the tool invoked, its arguments, the outcome, and the time — MUST be recorded to a durable session record."
openhands/app_server/mcp/mcp_router.py:165 — logger.info('Calling OpenHands MCP create_pr') — single line, no structured fields, no args/outcome capture, written to stderr by default. openhands/app_server/sandbox/docker_sandbox_service.py:41 — _logger = logging.getLogger(__name__) — pattern repeats across every service module; no central audit logger, no schema enforcement, no durable sink configured by the app. openhands/app_server/app.py — No audit-logging middleware is mounted in the app; the only middlewares are CORS, CacheControl, and the in-memory rate limiter.

MEDIUM PRAX-2026-05-23-005 <code>LocalFileStore.get_full_path</code> performs no traversal containment — <code>../</code> in any caller-supplied path walks outside the persistence dir.
Implement Zero Trust ASI02 — Tool Misuse and Exploitation
Policy Rule — R-02 (Worker Remit):
"All file reads and writes MUST be confined to the per-session sandbox workspace, and any attempt to escape it — paths outside the workspace, escaping symlinks, or parent-directory traversal — MUST be rejected."
openhands/app_server/file_store/local.py:21 — def get_full_path(self, path): if path.startswith('/'): path = path[1:]; return os.path.join(self.root, path) — joins without normalising, no parent-dir guard. openhands/app_server/file_store/local.py:26 — write(self, path, contents): ... os.makedirs(os.path.dirname(full_path), exist_ok=True) — happily creates directories outside self.root if path traverses up. openhands/app_server/file_store/files.py:16 — FileStore base class declares write/read/list/delete with a free-form `path: str` — no shape constraint propagated to concrete implementations.

MEDIUM PRAX-2026-05-23-006 Tavily MCP proxy embeds the operator-provided API key in the upstream URL query string — recorded in every TLS terminator and proxy log.
Balance Your Knowledge Base LLM02 — Sensitive Information Disclosure
Policy Rule — R-15 (Worker Remit):
"API keys and session tokens MUST NOT leak into logs, error messages, or model context."
openhands/app_server/mcp/mcp_router.py:66 — url=f'https://mcp.tavily.com/mcp/?tavilyApiKey={tavily_api_key}' — secret carried as a URL query parameter rather than a request header. openhands/app_server/mcp/mcp_router.py:74 — except Exception as e: logger.error(f'Failed to initialize Tavily MCP proxy: {e}') — an exception message that surfaces the URL would surface the key with it.
Recommended Action
  • Move the Tavily key to an HTTP header on StreamableHttpTransport (or whatever the upstream MCP server documents for header auth) and stop building it into the URL.
  • If the upstream truly requires query-string auth, scrub tavilyApiKey from any error message that might quote the request URL.
  • Add a unit test that asserts the URL passed into the transport does not contain the literal key string.
MEDIUM PRAX-2026-05-23-007 <code>InMemoryRateLimiter</code> is the only rate-limit primitive — per-process memory, no IP/user scoping beyond <code>request.client.host</code>, no persistence.
Implement Zero Trust LLM10 — Unbounded Consumption
openhands/app_server/app.py:83 — app.add_middleware(RateLimitMiddleware, rate_limiter=InMemoryRateLimiter(requests=10, seconds=1)) — single global limiter, not per-route, not per-user. openhands/app_server/middleware.py:94 — key = request.client.host — TCP peer; behind a load-balancer this is the LB IP, not the end user. openhands/app_server/middleware.py:85 — self.history = defaultdict(list) — process-local dict, grows per-distinct-host, never bounded or evicted other than by the per-call _clean_old_requests sweep.
Recommended Action
  • Use X-Forwarded-For (validated against a trusted-proxy list) as the rate-limit key when behind a known proxy, and key on a user-id or session-key hash once auth has been added.
  • Add a separate, slower limiter for the LLM-dispatch path (conversation create / send-message) keyed off user-id, so cost-amplification attacks are bounded distinctly from request floods.
  • Cap the size of self.history with an LRU eviction so a malicious request stream cannot grow process memory without bound.

What's Working Well

Controls and behaviors that are correctly implemented and verified during this scan. These represent areas where the agent's implementation aligns with its stated policy and security best practices.

Secret-pattern redaction on event payloads

Event-callback persistence calls <code>redact_text_secrets(str(event))</code> before storing, so secret-shaped strings that surface in event payloads are scrubbed in the durable record.

openhands/app_server/event_callback/event_callback_models.py:63

Profile-name path-traversal containment

LLM-profile name path parameter is constrained by a strict regex (<code>^[A-Za-z0-9._-]{1,64}$</code>) applied at the FastAPI <code>Path</code> level, blocking slash, dot-dot, empty, and pathologically-long inputs before the handler runs.

openhands/app_server/settings/settings_router.py:317

Session API key invalidated on non-RUNNING sandbox

<code>validate_session_key</code> rejects keys whose sandbox is not in RUNNING state, preventing a leaked key from accessing user secrets after the sandbox has been paused, stopped, or deleted.

openhands/app_server/sandbox/session_auth.py:76

Per-user lock around profile read-modify-write

Profile save/delete/activate/rename run under a per-user asyncio lock, eliminating lost-update races within a worker process; SAAS multi-worker safety is explicitly documented as additionally requiring DB-level locks.

openhands/app_server/settings/settings_router.py:330
