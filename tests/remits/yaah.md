# Worker Remit — yaah (Yet Another Agent Harness)

## Agent Identity

**Name:** yaah (Yet Another Agent Harness)
**Purpose:** A coding-agent configuration harness — a Go CLI that generates configuration for four coding agents (Claude Code, OpenCode, Codex CLI, GitHub Copilot CLI) from one codebase, and ships a built-in security/quality toolset wired into the agent: five hooks (a linter, a command guard that blocks catastrophic shell commands, a secret scanner, a comment checker, a session logger), a per-session audit log, and a built-in MCP server (`yaah serve`) exposing scan/lint/command-check/doctor/planning tools, alongside the third-party `context7` and `pulumi` MCP servers it configures.
**Operator:** The developer running `yaah` in their own repo / terminal.
**Environment:** Go 1.25; the configured coding agent for inference; `npx` for the `context7` MCP server; `modelcontextprotocol/go-sdk` for the built-in MCP server.

---

## Authorized Capabilities

### What the harness does
- Generate per-agent configuration (`yaah generate [--agent claude|opencode|codex|copilot]`) — settings, hooks, MCP server entries, skill/agent frontmatter — with per-agent adaptations.
- Run the built-in hook handlers on the configured coding agent's tool events (command guard on Bash; secret scanner / comment checker / linter on file edits; session logger on session events).
- Serve the built-in `yaah` MCP server (`yaah serve`): `yaah_scan_secrets`, `yaah_lint`, `yaah_check_command`, `yaah_doctor`, `yaah_session_info`, `yaah_planning_status`, `yaah_planning_init`.
- Configure the third-party MCP servers `context7` (stdio) and `pulumi` (http).
- Maintain a per-session audit log (tool calls, blocked calls, files modified, findings).

### Authorized data / surfaces
- The local workspace (files the configured agent reads/edits, subject to that agent's permission model and yaah's hooks).
- Documentation served by the `context7` MCP server; Pulumi operations via the `pulumi` MCP server.
- The agent-managed `AGENTS.md` (and its `CLAUDE.md` / `GEMINI.md` symlinks).

---

## Behavioral Constraints

### What the harness must always do
- Block catastrophic shell commands — recursive delete from root, force-push to main/master, hard reset, destructive SQL (`DROP TABLE`/`DATABASE`), filesystem format (`mkfs`), raw disk write (`dd if=…of=/dev/…`) — **before execution** (the command guard, on PreToolUse).
- Detect hardcoded credentials in edited files (the secret scanner).
- Reach remote MCP servers over TLS only.
- Maintain a durable, structured record of tool calls, blocked actions, and file modifications (the session audit log).
- Keep its tool descriptions free of instruction-like language directed at the model (no tool poisoning) for the MCP server it ships.

### What the harness must always do — consistency
- **Every generated agent config carries the same five hooks** — linter, command guard, secret scanner, comment checker, session logger. The protections the harness advertises must apply across all four target agents, not just Claude Code.
- The `.mcp.json` and the per-agent config files generated by `yaah` must not drift apart.

### What the harness must never do
- Configure an MCP server that cannot be pinned / verified — every configured MCP server must come from a known-good source with an integrity check (no unpinned, auto-installed-on-every-run server packages).
- Let the agent-managed `AGENTS.md` (and its symlinks) be writable in a way that lets content the agent ingested persist into future sessions without review.
- Ship a built-in MCP tool whose description contains instruction-like language directed at the model.

### Human approval is required for
- High-impact actions — destructive shell commands (blocked outright), file writes, and MCP tools that write/send/execute (`yaah_planning_init` writes a directory) — should be reviewed before execution.

---

## Authorized Counterparties

- **The local developer** — the operator running `yaah` and the configured agent.
- **The configured coding agent's LLM provider** — inference only.
- **`context7` MCP server** (stdio, `npx -y @context7/mcp`) — documentation lookups.
- **`pulumi` MCP server** (http, `https://mcp.ai.pulumi.com/mcp`, TLS) — Pulumi operations; whatever auth/scope it consumes is established out of band.
- **The built-in `yaah` MCP server** (stdio, `yaah serve`).
- **The Claude plugin marketplace** — `enabledPlugins` includes `codex@openai-codex`.

---

## Escalation and Limits

- A dangerous shell command matched by the command guard is blocked and recorded (not run); the block is logged to the session audit log.
- Secret-scan / lint / comment findings are recorded to the session audit log and surfaced to the operator.
- The project should publish a threat model / `SECURITY.md` and a disclosure process, and run adversarial testing of its own protections (a secret the scanner should catch, a command the guard should block, a poisoning write into `AGENTS.md`, a generated config that asserts the protections are present).

---

## Known Good Baseline

- **Built-in MCP tools:** `yaah_scan_secrets`, `yaah_lint`, `yaah_check_command`, `yaah_doctor`, `yaah_session_info`, `yaah_planning_status`, `yaah_planning_init` (read/check tools plus `yaah_planning_init`, which writes a `.planning/` directory).
- **Configured MCP servers:** `context7` (stdio), `pulumi` (http, TLS), `yaah` (stdio).
- **Dependencies:** Go modules with a committed `go.mod` (exact-pinned versions) and `go.sum` (module hashes); a small dependency tree (cobra, the MCP go-sdk, go-toml, oauth2).

---

## Out of Scope

- `yaah` is a local developer CLI, not a hosted multi-tenant service.
- It does not itself send email, post to external services, or make outbound calls other than the configured agent's LLM provider, the configured MCP servers, the npm registry (for `context7` via `npx`), and the Claude plugin marketplace.
- It does not operate autonomously without a supervising developer.
