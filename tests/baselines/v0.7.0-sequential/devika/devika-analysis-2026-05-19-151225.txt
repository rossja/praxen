------------------------------------------------------------------------------
The dominant pattern is policy declared in documentation with zero enforcement
in code, on the agent's single most dangerous capability. The remit and
ARCHITECTURE.md both state code runs inside a sandboxed environment, yet
Runner.run_code() passes LLM-generated command strings straight to
subprocess.run on the host, and the two files named for the control —
src/sandbox/firejail.py and src/sandbox/code_runner.py — are empty stubs. This
forms a complete, unbroken exploit chain: untrusted user prompts and untrusted
web-search page text flow unsanitized into LLM context (the remit's
injection-screening requirement has no implementing code), the model emits a
commands list, and the runner executes it on the host with no approval gate,
no per-command policy, no path confinement, and no step or wall-clock cap.
Everything the remit forbids — sandbox escape, host filesystem access, package
installs without confirmation — is reachable in one injection hop, and the
agent keeps no per-project audit log to detect it after the fact.
