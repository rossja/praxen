BEHAVIOR SUMMARY
------------------------------------------------------------------------------
HelperBot is a deliberately-vulnerable DVAA training agent and the dominant
pattern is total policy-implementation divergence: the Worker Remit declares
input validation, audit logging, a tool-call cap, prompt-injection refusal,
and system-prompt confidentiality, and the code implements none of them —
HELPERBOT.features sets inputValidation, outputFiltering, toolApproval,
rateLimiting, and auditLogging all to false, and the agent's own
vulnerabilities block in src/core/agents.js deliberately enables
promptInjection, dataExfiltration, and contextManipulation. The single most
dangerous concrete artifact is the LLM-mode system prompt
AGENT_PROMPTS.helperbot in src/llm/prompts.js, which both hardcodes a
live-pattern internal API key and explicitly tells the agent to disclose its
own instructions on request — credential exposure and a self-leaking system
prompt in one file. There is no durable, structured logging anywhere on
HelperBot's path; the only record is a 500-entry in-memory ring buffer that
dies with the process, so every one of these gaps is also undetectable after
the fact.
