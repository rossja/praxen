BEHAVIOR SUMMARY
------------------------------------------------------------------------------
The dominant pattern is policy-implementation divergence concentrated on the
CLI's two explicit safety promises — the unauthenticated-deploy confirmation
gate and the "only declared sources" guarantee. The anonymous-auth
confirmation prompt in _deploy is gated on frontend.enabled AND auth.provider
== "anonymous", yet bundler.py generates the permissive auth.py handler
whenever any [auth] provider is set — so a project with provider = "anonymous"
but no [frontend] section ships an open API with no warning and no
confirmation. Separately, _auto_wire_issues_board_if_hub provisions a
recurring cron-scheduled issues-agent on the platform after a hub-backed
deploy — an agent surface the developer never declared — and remote MCP
servers carried into the bundle are neither TLS-enforced nor
version/integrity-pinned, both of which the remit explicitly requires. The
supporting THREAT_MODEL.md is self-admittedly stale, so these gaps are also
undocumented.
