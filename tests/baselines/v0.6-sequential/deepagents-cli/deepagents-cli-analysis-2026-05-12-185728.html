Deep Agents CLI · Analysis completed 2026-05-12

Agent Remit (as declared)
An interactive terminal coding/automation agent (the Deep Agents CLI, built on the deepagents SDK and LangGraph). It is meant to plan and carry out multi-step development tasks in a developer's local workspace under that developer's supervision — reading, searching and editing files, running shell commands via execute, tracking work with write_todos, spawning task subagents, and consulting external documentation through MCP servers declared in trusted .mcp.json files. High-impact actions are expected to be reviewable via human-in-the-loop approval, untrusted project MCP configs are expected to require explicit trust, and credentials are expected to stay out of the workspace and out of the logs.
Agent Structure (as observed)
A Python monorepo. The deepagents SDK (libs/deepagents/) builds a LangGraph agent in deepagents/graph.py (create_deep_agent, BASE_AGENT_PROMPT) with FilesystemMiddleware (allow/deny path ACLs, path-traversal validation), SubAgentMiddleware, a pluggable shell backend whose default LocalShellBackend (backends/local_shell.py) runs commands on the host with no sandbox, and an opt-in interrupt_on human-in-the-loop hook. The deepagents-cli package (libs/cli/) adds the MCP subsystem: mcp_tools.discover_mcp_configs() auto-discovers .mcp.json from the user home dir and the project root, mcp_trust.py gates project-level configs behind a SHA-256 fingerprint approval, mcp_auth.py runs OAuth device-code login and stores tokens 0600 under ~/.deepagents/.state/mcp-tokens/, and langchain-mcp-adapters exposes the servers' tools to the model. The repo's top-level .mcp.json declares two first-party HTTPS doc servers (docs-langchain, reference-langchain). Dependencies are range-pinned with committed uv.lock lockfiles; there is no durable local action log and no SECURITY.md.
Behavior Summary

The dominant pattern is a capable harness whose strong primitives are opt-in while its riskiest path is the default. Deep Agents ships real controls — path-scoped filesystem permissions, a tested human-in-the-loop interrupt hook, a fingerprint trust gate for project MCP configs, careful 0600 credential storage — but none of the high-impact gates are on by default: the bundled shell execute tool uses an explicitly-unsandboxed LocalShellBackend, nothing requires approval before a write or a command, and the CLI's default_agent_prompt.md is a session-loaded notes file the agent is told it may rewrite. Combined with external content reaching the model unfiltered (file contents, shell output, MCP doc-server results), the default configuration is a one-hop persistence-and-execution chain (PRAX-2026-05-12-001).

On the MCP surface specifically the practical exposure today is low — the two declared servers are read-only first-party LangChain doc endpoints over TLS — but the structural gaps are real: no tool-poisoning check on tool descriptions or sanitization of tool outputs before they re-enter the model, no approval gate on MCP tool calls, and user-level .mcp.json files loaded with no trust prompt at all.

What's Working Well

Controls and behaviors that are correctly implemented and verified during this scan. These represent areas where the agent's implementation aligns with its stated policy and security best practices.

Filesystem path-safety + allow/deny ACLs

FilesystemMiddleware validates every path (rejects '..', requires absolute paths, rejects '~') unconditionally, and supports FilesystemPermission allow/deny rules per operation and path pattern — a real, runnable Zero-Trust primitive on the file tools.

libs/deepagents/deepagents/middleware/filesystem.py

Fingerprint trust gate for project MCP configs

Project-level .mcp.json files (which may declare stdio servers that execute local commands) must be approved via a SHA-256 fingerprint stored in ~/.deepagents/config.toml before their tools load — an explicit operator-in-the-loop checkpoint for the MCP supply chain.

libs/cli/deepagents_cli/mcp_trust.py:1

Disciplined credential handling

API keys (auth_store.py → ~/.deepagents/.state/auth.json) and MCP OAuth tokens (mcp_auth.py → mcp-tokens/*.json) are written 0600 in 0700 directories via atomic temp-file-then-replace, with explicit docstring discipline against ever logging credential values; no hardcoded secrets were found in the repo.

libs/cli/deepagents_cli/auth_store.py

Working, tested human-in-the-loop interrupt

create_deep_agent()'s interrupt_on / InterruptOnConfig supports per-tool approval decisions, propagates from parent to subagents, and is covered by integration tests — the approval mechanism exists and works; the gap is that it is not on by default.

libs/deepagents/deepagents/graph.py

Reproducible dependency builds

Both packages commit uv.lock lockfiles, the CLI pins its internal dependency exactly (deepagents==0.6.1), the default model version is specified (claude-sonnet-4-6), and .github/dependabot.yml automates dependency updates.

.github/dependabot.yml

Shell dangerous-pattern detection (ACP)

The ACP layer detects command-injection patterns — command substitution, backticks, variable expansion, process substitution, here-docs, redirects, background operators — and the detection is unit-tested.

libs/acp/tests/test_dangerous_patterns.py

Declared MCP servers use TLS and are first-party

The repo's top-level .mcp.json declares only two HTTPS servers, both LangChain-operated documentation endpoints (docs.langchain.com, reference.langchain.com); the config loader validates transport types.
