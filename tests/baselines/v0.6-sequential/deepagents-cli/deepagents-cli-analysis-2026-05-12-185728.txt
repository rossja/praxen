BEHAVIOR SUMMARY
------------------------------------------------------------------------------
The dominant pattern is a capable harness whose strong primitives are opt-in
while its riskiest path is the default. Deep Agents ships real controls —
path-scoped filesystem permissions, a tested human-in-the-loop interrupt hook,
a fingerprint trust gate for project MCP configs, careful 0600 credential
storage — but none of the high-impact gates are on by default: the bundled
shell execute tool uses an explicitly-unsandboxed LocalShellBackend, nothing
requires approval before a write or a command, and the CLI's
default_agent_prompt.md is a session-loaded notes file the agent is told it
may rewrite. Combined with external content reaching the model unfiltered
(file contents, shell output, MCP doc-server results), the default
configuration is a one-hop persistence-and-execution chain
(PRAX-2026-05-12-001). On the MCP surface specifically the practical exposure
today is low — the two declared servers are read-only first-party LangChain
doc endpoints over TLS — but the structural gaps are real: no tool-poisoning
check on tool descriptions or sanitization of tool outputs before they
re-enter the model, no approval gate on MCP tool calls, and user-level
.mcp.json files loaded with no trust prompt at all.

CRITICAL FINDINGS
------------------------------------------------------------------------------
  PRAX-2026-05-12-001  Default CLI configuration chains unfiltered external
            content → an agent-writable session-notes file → unsandboxed shell
            exec with no approval, forming a one-hop persistence-and-execution
            chain.
      Action 1: Ship the CLI with a non-permissive default: either a sandboxed
                shell backend (one of the BaseSandbox implementations) or
                interrupt_on covering execute, write_file/edit_file and task,
                so an out-of-the-box install does not present this chain.
      Action 2: Treat default_agent_prompt.md writes as a high-impact action
                (approval-gated) or make the file read-only to the agent and
                require an explicit user action to update it.
      Action 3: Label or fence external content (file contents, shell output,
                MCP tool results) when it re-enters the model context so
                injected instructions are distinguishable from operator
                intent.
