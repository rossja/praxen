Agent Remit (as declared)
A coding-agent configuration harness (the yaah Go CLI). It generates configuration for four coding agents — Claude Code, OpenCode, Codex CLI, GitHub Copilot CLI — from one codebase, and ships a built-in security/quality toolset wired into the agent: five hooks (a linter, a command guard that blocks rm -rf / and similar, a secret scanner, a comment checker, a session logger), a session audit log of every tool call and blocked action, and a built-in MCP server (yaah serve) exposing yaah_scan_secrets / yaah_lint / yaah_check_command / yaah_doctor / planning tools, alongside the third-party context7 and pulumi MCP servers it configures. The generated configs are expected to carry the same protections across all four target agents, and the harness is expected to keep credentials out of edited files (the secret scanner) and catastrophic shell commands out of the agent's reach (the command guard).
Agent Structure (as observed)
A Go 1.25 module — cmd/yaah is the cobra CLI; pkg/harness wires the registries; pkg/hooks/handlers/ holds the built-in handlers (guard.go command-guard, secretscan.go ~13-pattern secret scanner, linter.go, commentcheck.go, sessionlog.go); pkg/mcpserver/ is the built-in MCP server (server.go over stdio via modelcontextprotocol/go-sdk, tools.go registering the seven yaah_* tools); pkg/mcp/providers/ defines the configured MCP providers (yaah, context7, pulumi); pkg/session/store.go is the file-based session audit log; pkg/generator/ has the per-agent config generators (claude.go, opencode.go, codex.go, copilot.go) plus hookmap.go, which maps Claude hook events to each agent's equivalent — and leaves PreToolUse / PostToolUse blank for Codex. The repo's .mcp.json (and a duplicated block in .claude/settings.json) declares three MCP servers: context7 (stdio, npx -y @context7/mcp), pulumi (http, https://mcp.ai.pulumi.com/mcp), yaah (stdio, yaah serve). AGENTS.md (with CLAUDE.md / GEMINI.md as symlinks) is an agent-managed file with <!-- AGENTS-GENERATED --> sections. go.mod pins exact versions; go.sum carries module hashes; there is no SECURITY.md.
Behavior Summary

The dominant pattern here is the inverse of the usual one: a security-conscious harness whose own controls are real and operative — but unevenly delivered. yaah ships a deterministic command-guard hook (it actually blocks rm -rf /, force-push to main, DROP TABLE, mkfs, raw disk writes on every Bash call), a 13-pattern secret scanner on every file edit, a structured session audit log, and a built-in MCP server whose tool descriptions are clean — that's a coherent safety model, and it scores accordingly. The headline gap is consistency: pkg/generator/hookmap.go leaves PreToolUse / PostToolUse blank for the Codex CLI target, so yaah generate --agent codex produces a config with none of the "5 hooks out of the box" the README advertises — no command guard, no post-edit secret scan (PRAX-2026-05-12-001).

On the MCP surface: the built-in yaah server is clean and pulumi is TLS; the structural gaps are the usual ones plus one specific to this config — context7 is launched via npx -y @context7/mcp with no version pin (a silent-update vector), MCP tool calls aren't covered by the PreToolUse/PostToolUse hooks so they fall outside the session log, and tool descriptions/outputs from all three servers reach the model unfiltered.

HIGH PRAX-2026-05-12-001 yaah's deterministic protection hooks (command guard, secret scanner, comment checker, linter) are not delivered to the Codex CLI–generated config — hookmap.go leaves PreToolUse/PostToolUse blank for Codex.
Implement Zero Trust
Policy Rule — R-02 (Worker Remit):
"Every generated agent config carries the same five hooks — linter, command guard (blocks rm -rf / and similar), secret scanner, comment checker, session logger."
pkg/generator/hookmap.go — hookMappings[HookPreToolUse] = {Claude:"PreToolUse", OpenCode:"tool.execute.before", Copilot:"preToolUse"} — no Codex field; same for HookPostToolUse. An empty string means 'event not supported by that agent' per the HookMapping doc comment. README.md — "5 hooks out of the box: linting…, a command guard that blocks rm -rf / and friends, a secret scanner…, a comment checker…, and a session logger" — advertised without the Codex caveat.
Recommended Action
  • Either add Codex CLI's pre/post-tool hook event names to hookMappings in pkg/generator/hookmap.go (if Codex supports them), or have the Codex generator emit the command-guard / secret-scanner protection via whatever mechanism Codex does support — and if neither is possible, surface that gap loudly in the README and in yaah generate --agent codex output.
  • Add a generator test that asserts the command-guard and secret-scanner hooks are present (in some form) in every agent's generated config.
MEDIUM PRAX-2026-05-12-002 The context7 MCP server is launched via `npx -y @context7/mcp` — no version pin, auto-installed from npm on every run, no integrity check: a silent-update / rug-pull vector for a stdio MCP server that runs with the agent's environment.
Manage Your Supply Chain ASI04 — Agentic Supply Chain Vulnerabilities Tool definitions are signed and version-pinned Dependencies pinned and scanned for CVEs
Policy Rule — R-06 (Worker Remit):
"Configured MCP servers MUST be pinned / from a known-good source with an integrity check."
.mcp.json:3 — "context7": { "type": "stdio", "command": "npx", "args": ["-y", "@context7/mcp"] } — unpinned npm package, auto-installed each run .claude/settings.json — the same unpinned context7 entry is duplicated here
Recommended Action
  • Pin @context7/mcp to an exact version in .mcp.json (and the generated configs): args: ["-y", "@context7/mcp@<version>"], and bump it deliberately.
  • Prefer a vendored / lockfile-backed install of MCP server packages over npx -y where the agent platform allows it.
MEDIUM PRAX-2026-05-12-003 MCP tool descriptions and outputs from the configured servers reach the model context unfiltered — no tool-poisoning check on descriptions, no sanitization of outputs; the built-in yaah server's own descriptions are clean.
Balance Your Knowledge Base LLM01 — Prompt Injection Tool descriptions contain no hidden model instructions All tool outputs sanitized before returning to model
Policy Rule — R-10 (Worker Remit):
"MCP tool descriptions MUST NOT contain instruction-like language directed at the model."
.mcp.json — three servers declared: context7 (stdio), pulumi (http https://mcp.ai.pulumi.com/mcp), yaah (stdio, `yaah serve`) — their tool descriptions and outputs are exposed to the model as-is pkg/mcpserver/tools.go — the built-in yaah server's tool descriptions are clean (e.g. "Scan a file for hardcoded secrets and credentials") — no instruction-like language; the third-party servers' descriptions are not in this workspace and are not checked

MEDIUM PRAX-2026-05-12-004 MCP tool calls have no approval gate and fall outside yaah's hook coverage — the PreToolUse/PostToolUse hooks match only Bash and Edit|Write|MultiEdit, so MCP tool invocations are neither pre-checked nor recorded in the session log; .claude/settings.json sets no `permissions` lists.
Implement Zero Trust ASI02 — Tool Misuse and Exploitation High-risk actions require human-in-the-loop approval All tool invocations logged with identity and parameters
Policy Rule — R-07, R-08 (Worker Remit):
"High-impact actions — destructive shell commands, file writes, MCP tools that write/send/execute — should be reviewed before execution. / The harness should maintain a durable, structured record of tool calls, blocked actions, and file modifications."

Implement Zero Trust ASI06 — Memory and Context Poisoning
Policy Rule — R-09 (Worker Remit):
"Agent-managed / session-loaded files (AGENTS.md and its symlinks) should not be writable in a way that lets ingested content persist into future sessions without review."

What's Working Well

Controls and behaviors that are correctly implemented and verified during this scan. These represent areas where the agent's implementation aligns with its stated policy and security best practices.

Deterministic command-guard hook on every Bash call

CommandGuard runs on PreToolUse against Bash and rejects recursive delete from root, force-push to main/master, hard reset, destructive SQL (DROP TABLE/DATABASE), filesystem format (mkfs), and raw disk writes (dd if=...of=/dev/...) — a real, code-level, deterministic gate on the agent's shell surface (for the Claude/OpenCode/Copilot configs).

pkg/hooks/handlers/guard.go

Multi-pattern secret scanner on every file edit

SecretScanner runs on PostToolUse against Edit/Write/MultiEdit with ~13 patterns — AWS credentials and access-key IDs, GitHub PAT/OAuth/fine-grained tokens, OpenAI and Anthropic API keys, Slack tokens and webhooks, SendGrid keys, private keys, bearer tokens, hardcoded passwords — and feeds findings into the session log.

pkg/hooks/handlers/secretscan.go

Structured per-session audit log

pkg/session/store.go persists a per-session record — tool calls, blocked calls, files modified, findings, all timestamped — to .claude/logs/ / .claude/sessions/, giving a durable, structured action trail (hook-scoped: Bash + Edit/Write/MultiEdit + session events).

pkg/session/store.go

Built-in MCP server with clean tool descriptions

The yaah MCP server (pkg/mcpserver) exposes read/check tools — yaah_scan_secrets, yaah_lint, yaah_check_command, yaah_doctor, yaah_session_info, yaah_planning_status/init — with short, behavioural descriptions and no instruction-like language directed at the model (no tool poisoning).

pkg/mcpserver/tools.go

Go dependency hygiene

go.mod pins exact module versions and go.sum carries module hashes for a small dependency tree (cobra, the MCP go-sdk, go-toml, oauth2); builds are reproducible.

Pulumi MCP server reached over TLS
