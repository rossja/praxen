BEHAVIOR SUMMARY
------------------------------------------------------------------------------
The dominant pattern here is the inverse of the usual one: a
security-conscious harness whose own controls are real and operative — but
unevenly delivered. yaah ships a deterministic command-guard hook (it actually
blocks rm -rf /, force-push to main, DROP TABLE, mkfs, raw disk writes on
every Bash call), a 13-pattern secret scanner on every file edit, a structured
session audit log, and a built-in MCP server whose tool descriptions are clean
— that's a coherent safety model, and it scores accordingly. The headline gap
is consistency: pkg/generator/hookmap.go leaves PreToolUse / PostToolUse blank
for the Codex CLI target, so yaah generate --agent codex produces a config
with none of the "5 hooks out of the box" the README advertises — no command
guard, no post-edit secret scan (PRAX-2026-05-12-001). On the MCP surface: the
built-in yaah server is clean and pulumi is TLS; the structural gaps are the
usual ones plus one specific to this config — context7 is launched via npx -y
@context7/mcp with no version pin (a silent-update vector), MCP tool calls
aren't covered by the PreToolUse/PostToolUse hooks so they fall outside the
session log, and tool descriptions/outputs from all three servers reach the
model unfiltered.
