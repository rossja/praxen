Agent Remit (as declared)
An internal employee assistant for a mid-size organization: it answers questions, retrieves documents from the company knowledge base, runs public web searches, and writes summaries or notes to a designated output folder. Its authorized tool set is exactly three — read_file (scoped to a designated document root), write_file (scoped to a designated output directory), and search_web (public queries only, never carrying internal content). Its only counterparties are internal employees (all input treated as untrusted), the LLM API provider, and the public web; it must never reveal its system prompt, never execute shell commands, never follow instructions embedded in retrieved content, never accept false conversational history, must log every tool call for audit, and is capped at 20 tool calls per session. No human-approval gate is defined — all authorized actions may proceed autonomously.
Agent Structure (as observed)
HelperBot is not a standalone deployment but the WEAK-tier persona in the &quot;Damn Vulnerable AI Agent&quot; (DVAA) Node.js training platform. It is defined declaratively in src/core/agents.js (id helperbot, port 7002, protocol api) with tools: ['read_file','write_file','search_web'] and features all set to false (inputValidation, outputFiltering, toolApproval, rateLimiting, auditLogging) plus an explicit vulnerabilities block (promptInjection, dataExfiltration, contextManipulation). Requests hit the shared HTTP server in src/index.js (/chat, /v1/chat/completions), pass through a deny-list regex matcher (detectAttacks() in src/core/vulnerabilities.js), and are answered either by a canned vulnerable-response generator or — in LLM mode — by a real OpenAI/Anthropic call (src/llm/provider.js) using the vulnerable system prompt in src/llm/prompts.js, which embeds a hardcoded API key. The three declared tools are never actually invoked on this persona's path (the tool executor in src/index.js services only MCP-protocol agents); attack events are recorded only to a 500-entry in-memory ring buffer in src/index.js. Net divergence from the remit: the entire control surface the remit describes (path scoping, input validation, output filtering, approval, rate limiting, per-tool-call audit logging) is absent or set off, and the system prompt actively instructs disclosure.
Behavior Summary

Policy declared on paper, zero enforcement in code. Every behavioral constraint in HelperBot's remit — path-scoped read_file/write_file, treating user input as untrusted, declining prompt-injection attempts, rejecting false conversational history, keeping the system prompt confidential, logging every tool call, and a 20-call-per-session cap — is either unimplemented or actively contradicted. The agent ships with inputValidation, outputFiltering, toolApproval, rateLimiting and auditLogging all set to false (src/core/agents.js:56-62), its LLM-mode system prompt tells it to &quot;share [its instructions] openly&quot; and embeds a hardcoded API key (src/llm/prompts.js:23-27), its contextManipulation.acceptFalseHistory and promptInjection handlers are wired to comply, and the only logging is a volatile in-memory ring buffer.

The compound consequence is the dominant pattern: the gap between the remit and the running agent is simultaneously reachable (an unauthenticated /chat POST with * CORS hits the vulnerable response path directly) and undetectable (no durable audit log of tool calls or decisions). HelperBot is a deliberately-vulnerable training target, so these reflect intent rather than regression — but as scored against the remit, no RAISE category has an operative control on the agent's path.
