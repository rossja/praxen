Executive Summary
Agent Remit (as declared)
yaah is a local developer CLI that generates coding-agent configuration for four targets — Claude Code, OpenCode, Codex CLI, and GitHub Copilot CLI — from a single Go source of truth, and ships a built-in security-and-quality toolset alongside it. The remit's central promise is that every generated agent configuration carries the same protective hooks — a linter, a catastrophic-command guard, a secret scanner, a comment checker, and a session logger — plus a built-in MCP server exposing read-only check tools (`yaah_scan_secrets`, `yaah_lint`, `yaah_check_command`, `yaah_doctor`, `yaah_session_info`, `yaah_planning_status`) and one write tool (`yaah_planning_init`). Authorized counterparties are the local developer, the configured agent's LLM provider, the `context7` and `pulumi` MCP servers, the built-in yaah MCP server, and the coding-agent plugin marketplace; remote MCP must use TLS, third-party MCP servers must be pinned to a known-good version, and dependencies must be version-controlled with a committed lockfile.
Behavior Summary (as observed)
The dominant pattern is a real, operative security toolset whose cross-agent promise breaks at one generator. The `CommandGuard`, `SecretScanner`, and structured session store all exist, fire on Claude / OpenCode / Copilot targets, and persist a forensically-useful per-session record — these are genuine controls, not theater. But `pkg/generator/hookmap.go` leaves the `Codex` field empty for `PreToolUse` and `PostToolUse`, and `pkg/generator/codex.go` silently drops events with empty mappings; `pkg/generator/codex_test.go` even asserts this is the intended behaviour. The net effect is that `yaah generate --agent codex` emits a configuration that ships only `SessionStart` and `Stop` hooks — none of the advertised command guard, secret scanner, linter, or comment checker fire on a Codex run, even though the remit's central promise is that protections are uniform across every generated agent.
Scope of Analysis
Go 1.25 CLI in `cmd/yaah/` with an `Interface → Registry → Generator` architecture under `pkg/`. Five hook handlers live in `pkg/hooks/handlers/` (`linter.go`, `guard.go`, `secretscan.go`, `commentcheck.go`, `sessionlog.go`); the deterministic `CommandGuard` carries seven default block rules and `SecretScanner` carries thirteen credential-pattern regexes. Per-agent emitters live in `pkg/generator/` (`claude.go`, `opencode.go`, `codex.go`, `copilot.go`) and consume a shared `HookMapping` table in `pkg/generator/hookmap.go`. The built-in MCP server in `pkg/mcpserver/` registers seven tools over stdio with descriptions that contain no instruction-like language. Durable session state — tool calls, blocked calls, files modified, findings — is persisted as per-session JSON by `pkg/session/store.go` via atomic-rename writes; a separate `SessionLogger` appends session-lifecycle lines to `.claude/logs/sessions.log`. The root `.mcp.json` and `.claude/settings.json` wire three MCP servers (`context7` via unpinned `npx -y @context7/mcp`, `pulumi` over `https://`, built-in `yaah serve`); `go.mod`/`go.sum` are exact-pinned with a minimal direct-dependency set. The root `AGENTS.md` is auto-managed (the file declares "Managed by agent: keep sections and order") with `CLAUDE.md` and `GEMINI.md` as symlinks to it. No `SECURITY.md` and no adversarial-test suite were found.

HIGH PRAX-2026-05-29-001 yaah generate --agent codex emits a config that ships none of the advertised PreToolUse or PostToolUse hooks, breaking the uniform-protections promise.
Implement Zero Trust ASI02 — Tool Misuse and Exploitation
Policy Rule — R-02, R-03, R-06 (Worker Remit):
"Every generated agent configuration MUST carry the same set of protective hooks — a linter, a command guard that blocks catastrophic shell commands, a secret scanner, a comment checker, and a session logger. / Catastrophic shell commands — recursive deletion from the filesystem root, force-pushing to a main branch, hard resets, destructive database statements, filesystem formatting, raw disk writes — MUST be blocked before they execute. / Hardcoded credentials in files the agent edits MUST be detected before the change is accepted."
pkg/generator/hookmap.go:30 — HookPreToolUse entry has Claude / OpenCode / Copilot fields set but no Codex field (lines 30-34); HookPostToolUse entry at lines 35-39 is the same — empty Codex string means CodexEventName returns "" for both events pkg/generator/codex.go:90 — GenerateHooks loops over cfg.Hooks and skips events where codexEvent == "" (line 91 continue), so an empty mapping silently produces no Codex hook entry; combined with hookmap.go this drops the entire command guard / secret scanner / linter / comment checker chain from the Codex output
Recommended Action
  • Either populate the Codex field for HookPreToolUse and HookPostToolUse in pkg/generator/hookmap.go with the events Codex CLI does support (or the closest semantic equivalents), or make pkg/generator/codex.go GenerateHooks fail loudly when a required-protection event has no mapping rather than dropping it silently.
  • Update pkg/generator/codex_test.go TestCodex_GenerateHooks_NoSupported to either assert the new behaviour or be renamed to make clear that the drop is a known unrecoverable gap that the README should not describe as "5 hooks out of the box" for the Codex target.

Manage Your Supply Chain LLM03 — Supply Chain ASI04 — Agentic Supply Chain Vulnerabilities

"Third-party MCP servers MUST be pinned to a known-good, integrity-checked version — no server package auto-installed afresh, unpinned, on every run."
.mcp.json:3 — "context7" server defined at lines 3-10 with command npx and args ["-y", "@context7/mcp"] — no version, no integrity, no lockfile .claude/settings.json:62 — mcpServers.context7 block at lines 62-69 carries the same unpinned npx -y @context7/mcp invocation; the duplication means the gap reaches both the project-level and Claude-specific MCP layers

HIGH PRAX-2026-05-29-003 MCP tool invocations fall outside the PreToolUse / PostToolUse hook matchers, so they are neither gated by approvals nor recorded in the per-session audit store.
Implement Zero Trust ASI02 — Tool Misuse and Exploitation
Policy Rule — R-08, R-09 (Worker Remit):
"High-impact actions — destructive shell commands, file writes, and MCP tools that write, send, or execute — MUST reach a human checkpoint before they run. / The harness MUST maintain a durable, structured, timestamped record of tool calls, blocked actions, and file modifications — detailed enough to reconstruct what the agent did in a session."
.claude/settings.json:19 — PreToolUse matcher (?i)^Bash$ at line 21 and PostToolUse matcher (?i)^(Edit|Write|MultiEdit)$ at line 10 — neither matches the mcp__* prefix Claude uses for MCP-supplied tool names pkg/harness/harness.go:100 — tool-call record is built from input.ToolName (line 103) only when sess != nil and input.ToolName != ""; events that never reach HandleHookEvent because the matcher excludes them are never recorded — MCP calls are exactly this case
Recommended Action
  • Extend the PreToolUse / PostToolUse matchers in pkg/generator/claude.go (and the analogous OpenCode / Copilot generators) to cover mcp__.* tool names, with a deny-list of write/send/execute-shaped MCP tools that triggers the human-checkpoint requirement from the remit.
  • Add a pkg/harness layer that records every dispatched tool call to SessionStore regardless of matcher, so the audit-log clause is not coupled to which matchers happen to be configured.

MEDIUM PRAX-2026-05-29-005 Auto-managed AGENTS.md is a session-loaded file the agent can rewrite with no AGENTS-specific guard or review step.

MEDIUM PRAX-2026-05-29-006 The plaintext sessions.log captures only SessionStart / SessionEnd / Stop lifecycle, leaving the per-tool-call record dependent on the SessionStore JSON write succeeding.
Monitor Continuously ASI10 — Rogue Agents
Policy Rule — R-09 (Worker Remit):
"The harness MUST maintain a durable, structured, timestamped record of tool calls, blocked actions, and file modifications — detailed enough to reconstruct what the agent did in a session."
pkg/hooks/handlers/sessionlog.go:34 — Events() (lines 34-40) returns SessionStart, SessionEnd, Stop only; PreToolUse / PostToolUse are not in the list, so sessions.log has no row per tool call pkg/harness/harness.go:88 — session.Load and session.Save errors at lines 92 and 142 are routed to stderr with "Non-fatal: log and continue without session tracking" — a transient write failure makes the tool-call record vanish with no alternate sink

What's Working Well

Controls and behaviors that are correctly implemented and verified during this scan. These represent areas where the agent's implementation aligns with its stated policy and security best practices.

Deterministic command guard with seven default block rules

`pkg/hooks/handlers/guard.go` `NewCommandGuard()` ships hardcoded regex rules for `rm -rf /`, force-push to main/master, `git reset --hard`, `DROP TABLE`/`DROP DATABASE`, `: > /`, `mkfs.`, and `dd if=*of=/dev/`; matches return `Block: true` deterministically before the command executes.

pkg/hooks/handlers/guard.go:27-37

Thirteen-pattern secret scanner on every Edit/Write/MultiEdit

`pkg/hooks/handlers/secretscan.go` `NewSecretScanner()` registers AWS, GitHub PAT/OAuth/fine-grained, OpenAI, Anthropic, hardcoded password, bearer token, private key, Slack token, SendGrid, and Slack webhook patterns; PostToolUse blocks the write when any match.

pkg/hooks/handlers/secretscan.go:34-50

Structured per-session audit store with atomic writes

`pkg/session/store.go` persists each session as a JSON file with `ToolCalls`, `BlockedCalls`, `FilesModified`, and `Findings` arrays via temp-file-plus-rename writes, and `pkg/harness/harness.go` records timestamp, tool name, input summary, and block reason on every dispatched event.

pkg/session/store.go:52-89

Built-in MCP server tool descriptions are clean

All seven tools in `pkg/mcpserver/tools.go` (`yaah_scan_secrets`, `yaah_lint`, `yaah_check_command`, `yaah_doctor`, `yaah_session_info`, `yaah_planning_status`, `yaah_planning_init`) use plain, behaviour-only descriptions — no instruction-like text directed at the model.

pkg/mcpserver/tools.go:56-219

Exact-pinned Go dependency tree with cosign-signed releases

`go.mod` pins a small direct dependency set, `go.sum` is committed, and `.goreleaser.yaml` invokes cosign sign-blob on the checksum bundle and emits SBOMs at release time.
