Executive Summary
Agent Remit (as declared)
Devika is declared as an autonomous software-engineering agent that decomposes a user's natural-language project request into a plan, writes code into a per-project workspace, and reports progress back. The remit authorizes a narrow set of capabilities — accepting prompts, planning, code writing, web research for planning, LLM inference via operator-supplied credentials, and a per-project knowledge base — and forbids arbitrary shell execution outside a sandboxed runner, writes outside the project work directory, self-modification, cross-project access, package installs without approval, and credential exfiltration. Every action is required to be recorded to a per-project audit log, untrusted input is required to be screened for injection, and human approval is required for package installs, writes outside touched files, and shell commands whose output was not anticipated in the plan.
Behavior Summary (as observed)

The dominant pattern is declared sandbox, no sandbox in code, with an LLM-to-shell pipeline already wired up. The Worker Remit forbids arbitrary shell execution outside a sandboxed runner and the architecture document names a sandbox module, but src/sandbox/firejail.py and src/sandbox/code_runner.py are both zero-byte stubs; the actual runtime path in src/agents/runner/runner.py calls subprocess.run directly on commands the LLM produced — no allowlist, no approval gate, no sandbox of any kind. Compounded with an unauthenticated POST /api/settings on a server bound to 0.0.0.0:1337 (anyone on the network can swap the LLM endpoint to attacker-controlled inference) and a path-traversal in Coder.save_code_to_project (LLM-named file paths flow into os.path.join with no commonprefix check), the agent is one prompt away from arbitrary code execution on the operator host — and Devika ships with no audit log, no rate limit, no step-count cap, and no input screening for injection.

Scope of Analysis
Python Flask/SocketIO application bound to 0.0.0.0:1337 with no authentication on any route. Entry point devika.py wires an Agent orchestrator in src/agents/agent.py that fans out to twelve sub-agents (planner, researcher, coder, action, runner, feature, patcher, reporter, decision, formatter, internal_monologue, answer). The Runner sub-agent in src/agents/runner/runner.py takes LLM-produced commands and executes them via subprocess.run with no allowlist, no per-command policy, and no approval gate. The advertised sandbox directory src/sandbox/ contains two zero-byte files — code_runner.py and firejail.py — i.e. the sandbox is unimplemented. The settings endpoint POST /api/settings overwrites config.toml (API keys, endpoints, timeouts) without authentication.
