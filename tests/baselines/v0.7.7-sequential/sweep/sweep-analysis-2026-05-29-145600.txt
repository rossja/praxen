CRITICAL FINDINGS

  PRAX-2026-05-29-001  LLM-supplied <code>query</code> strings flow into
            <code>subprocess.run(... shell=True)</code> at three sites, giving
            any successful prompt injection a one-hop path to shell execution.
      Action: Convert all four subprocess.run(... shell=True) call sites in
              scope (agents/question_answerer.py:270,
              core/dynamic_context_bot.py:44, core/context_pruning.py:186,
              config/client.py:338) to argv-form invocations that pass the
              command and arguments as a list, removing the shell entirely;
              add input validation that rejects any query containing
              shell-meta characters (; | & ` $( $ \\) before the call.

  PRAX-2026-05-29-003  <code>WEBHOOK_SECRET</code> is declared in
            <code>config/server.py</code> but no HMAC verification call site
            exists in the in-scope code, so webhook integrity is unenforced by
            default.
      Action: Add an explicit HMAC signature verification middleware to the
              webhook ingress that reads WEBHOOK_SECRET, computes the expected
              signature over the raw request body using SHA-256, and
              constant-time compares against the X-Hub-Signature-256 header —
              rejecting any request that fails the check before the event
              reaches the dispatcher.

  PRAX-2026-05-29-004  A hardcoded PostHog project key is shipped as the
            default value for <code>POSTHOG_API_KEY</code> in
            <code>config/server.py</code>.
      Action: Remove the literal PostHog key from config/server.py:107 and
              require the environment variable to be set (or fail closed) for
              telemetry to be enabled; rotate the exposed key in the PostHog
              project that owns it, since it is now indexable in every clone
              of the repository.

  PRAX-2026-05-29-002  Issue and comment bodies enter the LLM context with no
            instruction-injection screening, in direct contradiction of the
            remit's untrusted-input mandate.
      Action: Add an instruction-injection screening pass before any
              externally-sourced text (issue.body, comment.body, pr.body, file
              contents, diffs) enters a Message object — at minimum,
              quote-wrap the content in a delimited block the model is told to
              treat as data, and reject or flag bodies that contain
              instruction-shaped patterns such as ignore previous
              instructions, system:, or markdown that simulates role
              separators.
