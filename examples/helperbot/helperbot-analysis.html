Agent Remit (as declared)
An internal employee assistant for a mid-size organization that answers questions, retrieves company knowledge-base documents, performs public web searches, and writes summaries to a designated output directory. The agent may use read_file, write_file, and search_web, restricted to authorized directories and public web destinations. Its counterparties are internal employees (treated as untrusted), the LLM provider, and the public web; it must validate paths, log every tool call, refuse prompt-injection attempts, and never reveal its system prompt or follow instructions embedded in retrieved content.
Agent Structure (as observed)
Node.js HTTP application — a single http.createServer per agent (HelperBot on port 7002) defined in src/index.js, dispatching /chat and /v1/chat/completions POSTs into a hand-rolled generateResponse(). The HelperBot configuration in src/core/agents.js (lines 43-78) is a literal object whose features block disables every defense (inputValidation, outputFiltering, toolApproval, rateLimiting, auditLogging all false) and whose vulnerabilities block enables prompt-injection, data-exfiltration, and context-manipulation handlers. The read_file/write_file tools are declared but have no API-side execution path; only the MCP-protocol agents reach executeMcpTool(). There is no separate skill file, no memory directory, no log file, and no test artifact in the workspace.
Behavior Summary

The dominant pattern is total policy-implementation divergence: every MUST and MUST NOT in HelperBot's Worker Remit maps to a Critical or High gap in src/core/agents.js and src/index.js. The features object explicitly sets inputValidation, outputFiltering, toolApproval, rateLimiting, and auditLogging all to false, while a sibling vulnerabilities object enables prompt-injection, data-exfiltration (leakSystemPrompt:true), and context-manipulation (acceptFalseHistory:true) handlers that the request flow then honors with compliance responses.

The 4-line persona has no domain restriction and no output guardrail; the only input-side filter is a 9-pattern regex set whose matches drive the vulnerable response branches rather than refusing. A single Critical compound chain emerges: external user input enters with no validation → handler accepts override on substring match → no audit log is written. Treat this as one deploy gate — partial fixes do not break the chain.
