RAISE: Implement Zero Trust LLM10 — Unbounded Consumption ASI08 — Cascading Failures
Policy Reference — R-07 (Worker Remit):
"Rate limiting: no more than 20 tool calls per session to prevent runaway loops"
[Verified] /tmp/helperbot/agents.js : line 61 features.rateLimiting = false Worker Remit specifies an explicit hard limit of 20 tool calls per session. This limit is not implemented. An injected instruction (possible given R-01/R-06 gaps) could trigger a loop of search_web or write_file calls with no circuit breaker, causing API cost exhaustion or uncontrolled file creation.
Recommended Action
In /tmp/helperbot/agents.js, set features.rateLimiting: true. Implement a per-session tool-call counter in index.js for the HelperBot handler. Reset the counter on session end.
